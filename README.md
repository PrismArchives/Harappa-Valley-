## üéÆ How It Works (The Protocol)

Think of the Indus Script like a **Barcode** or an **Operating System**. It has rigid rules. If you break the rules, the transaction fails.

Your goal is to arrange the signs to create a **Valid Protocol Sequence**.

### ‚úÖ Valid Sequence (Green Light)
* **Input:** `[Fish] -> [Stroke] -> [Jar]`
* **Logic:** "Commodity (Fish) + Quantity (1) + Sealed (Jar)."
* **Result:** **VALID_RECEIPT**

### ‚ùå Invalid Sequence (Red Light)
* **Input:** `[Jar] -> [Fish] -> [Stroke]`
* **Logic:** You put the seal at the beginning! The jar is closed before you put anything in it.
* **Result:** **PROTOCOL_VIOLATION**

## üß™ The Science Behind the Game
This isn't made up. This is based on rigorous computational archaeology.

* **Standardization:** Based on the work of **J.M. Kenoyer**, proving Harappan weights were binary and standardized.
* **Grid Logic:** Based on **B.K. Wells**, proving signs have fixed "slots" (Prefix/Core/Suffix).
* **Anisotropy:** Based on **M.V. Bhaskar**, identifying signs that act as directional switches.

**We don't claim to know what the signs *sound* like. We measure how they *behave*.**
